# CSTO v2 (`csto2`) Log & Artifact Guide

This guide describes the output files and directories generated by the CSTO v2 pipeline, and how to use them for debugging.

---

## 1. Directory Structure

All pipeline outputs are stored in `.csto2/` (mapped to `/home/gcohen4/otog/results/` on the host):

```text
.csto2/
├── config.properties        # Persisted run configuration (Git URL, commits, JVM paths, etc.)
├── deps.classpath           # Autodetected target classpath entries
├── tests.all                # Complete list of discovered test classes in the module
├── tests.runnable           # Runnable subset of test classes (filtered)
├── trace/                   # Tracing/calibration phase outputs
│   ├── trace.jsonl          # Merged test timings + JVM agent facts
│   ├── orders/              # Test order files (.order) sent to Surefire
│   ├── agent/               # Raw JVM metrics (.facts.jsonl) from JFR/MXBeans
│   └── logs/                # Raw console logs (.log) from Maven Surefire runs
├── select/                  # Candidate selection phase outputs
│   ├── *.order              # Generated test orders for each candidate strategy (the shipped winner is printed as `=> SHIP: <strategy_name>`)
│   ├── measure.jsonl        # Raw timing results for candidate selection rounds
│   ├── select-report.log    # Saved console report block (measurements + Wilcoxon test if run in scientific mode)
│   └── logs/                # Raw console logs for candidate runs (e.g., rt-heavy-tail_3.log)
```

---

## 2. Key Log Files & Formats

### Maven Surefire Console Logs (`logs/*.log`)
* **Paths**: `trace/logs/*.log` or `select/logs/*.log`
* **Format**: Plain text
* **What to use them for**: 
  These capture the raw stdout/stderr of the Maven Surefire processes. Open these logs first if a run fails with `exit=1` or crashes. Look for compilation errors, missing dependencies, or test failures.
  * *Note*: Select logs are sanitized (e.g., the round identifier `#3` in `initial#3` becomes `initial_3.log` on disk).

### Config Properties (`config.properties`)
* **Path**: `config.properties`
* **Format**: Plain text key-value properties
* **What to use them for**:
  Inspect or manually override configuration options (e.g., overriding Maven command options (`mvnopts`), custom java paths, or approaches to skip).

### Master Trace timings (`trace/trace.jsonl`)
* **Path**: `trace/trace.jsonl`
* **Format**: JSON Lines
* **What to use them for**:
  Verify the calibrated per-class metrics compiled by the JFR JVM agent (`classesLoaded`, `jitMs`, `gcCount`, `gcMs`, `allocBytes`).

### Selection Performance timings (`select/measure.jsonl`)
* **Path**: `select/measure.jsonl`
* **Format**: JSON Lines
* **What to use them for**:
  Compare raw execution times for each candidate strategy across all rounds to see why a candidate was picked or disqualified.

## 3. How to Debug Common Failures

### 1. Surefire runs fail with no reports
* **Symptom**: Dynamic trace crashes with `java.lang.IllegalStateException: trace produced no rows`.
* **Fix**: Open `trace/logs/initial.log`. It is typically a compilation error, maven dependency resolution error, or config mismatch.

### 2. Test Flakiness (Red Tests)
* **Symptom**: A candidate run fails due to order-dependent tests.
* **Fix**: Look up the failing class name in the corresponding `.log` file under `select/logs/` to find the exact run order in `select/*.order`.

### 3. Corrupted native channel warnings
* **Symptom**: `Corrupted channel by directly writing to native stream in forked JVM` in logs.
* **Fix**: Usually benign. If a crash occurs, search for binary outputs or `System.out` prints immediately before the warning.
